# Custom GPT-2 Architecture & Streaming Web App 🚀

This repository contains a PyTorch implementation of a GPT-2 Large Language Model (LLM) built from scratch. It includes a custom training loop, a beautiful Light Mode web interface with real-time SSE streaming, and an OpenAI-compatible REST API. 

## Technical Model Specifications
This project implements the exact architecture of the **GPT-2 Small (124M Parameters)** model based on the original paper:
- **Parameter Count:** 124 Million
- **Transformer Layers:** 12
- **Attention Heads:** 12
- **Embedding Dimension:** 768
- **Context Window Size:** 1024 tokens
- **Activation Function:** GELU

## Training Details & Reproducibility
The repository includes a custom training script (`gpt_archticture.py`) built to train the model from scratch.
- **Dataset:** Tiny Shakespeare (or custom text files)
- **Hardware:** CPU or NVIDIA GPU (CUDA automatically detected)
- **Epochs:** 4
- **Batch Size:** 4
- **Context/Stride:** 256 max length, 128 stride during initial training phases
- *Note:* The final web application uses pre-trained weights for demonstration, but you can swap them out for weights generated by your own training loop!

## Prerequisites & System Requirements
- **Python:** 3.9+
- **RAM:** Minimum 4GB (8GB recommended)
- **GPU:** Optional, but a CUDA-compatible NVIDIA GPU is highly recommended for training.
- **Dependencies:** `torch`, `flask`, `tiktoken`, `requests`, `tensorflow-cpu` (for reading OpenAI weight formats).

## Quick Start

1. Clone the repository:
   ```bash
   git clone https://github.com/Manindra-babu/MiniGpt.git
   cd MiniGpt
   ```
2. Install the required packages:
   ```bash
   pip install -r requirements.txt
   ```
3. Run the server:
   ```bash
   python app.py
   ```
4. **Interact with the Web UI:** Open your browser and navigate to `http://localhost:5000`. You will be greeted by a modern, clean chat interface. You can type a prompt or click one of the preset suggestion chips. The response will stream onto your screen character-by-character in real-time!

## Environment Variables
If you are deploying this to the cloud (e.g., Hugging Face Spaces or Render), you can configure the following environment variables:
- `API_TOKEN`: The secret key used to authenticate API requests. (Default: `sk-custom-llm-token`)
- `PORT`: The port the Flask server binds to. (Default: `5000`)

## Pre-Trained Weights
You do **not** need to manually download weights or train from scratch to use the app. When you run `app.py` for the first time, the script will automatically reach out to OpenAI's public servers, download the official 124M GPT-2 weights, and map them tensor-by-tensor into our custom PyTorch architecture.

## API Drop-In Replacement (Python Example)
Because the API perfectly mirrors OpenAI's `/v1/completions` schema, you can easily plug your custom LLM into any existing Python project:

```python
import requests

# Point this to your localhost or your cloud deployment URL
api_url = "http://localhost:5000/v1/completions"

headers = {
    "Authorization": "Bearer sk-custom-llm-token",
    "Content-Type": "application/json"
}

data = {
    "prompt": "Question: What is the meaning of life?\nAnswer:",
    "max_tokens": 150,
    "temperature": 0.8
}

response = requests.post(api_url, headers=headers, json=data)

if response.status_code == 200:
    print(response.json()["choices"][0]["text"])
else:
    print("Error:", response.text)
```

## Known Limitations
- **Model Size:** This is a 124M parameter model. It does not have the reasoning capabilities of massive 70B+ models.
- **Instruction Tuning:** The model relies on pre-trained completion weights rather than strict instruction-tuning. The backend secretly wraps user prompts in a `Question: ... \nAnswer:` format to encourage question-answering behavior.
- **Inference Speed:** Running on CPU (e.g., Hugging Face Free Tier) takes several seconds, though this is heavily masked by the Server-Sent Events (SSE) streaming UI.
